1 — CSS Inclusion Methods

Method 1 — Inline CSS

This paragraph uses the style="" attribute directly. Works, but avoid in production.

⚠️ Inline styles have the highest specificity of all (short of !important), making them very hard to override later.

Method 2 — Internal CSS

This paragraph is styled by an internal <style> tag in the <head> of this HTML file.

⚠️ Internal styles apply to only this page and are not cached by the browser separately.

Method 3 — External CSS ✅ Recommended

This entire page is styled via an external phase1-foundations-typography.css file, linked in the <head>. This is the standard for all real projects.

2 — Basic Selectors

Element Selector — p {}

This paragraph is targeted by the element selector p. Every paragraph on the page shares this base style.

So does this one — element selectors are broad and low-specificity.

Use element selectors for site-wide defaults (like resetting margins or setting base font size). Avoid for component-level styling.

Class Selector — .class-name {}

Classes are the recommended way to style almost everything. They are reusable (.btn appears on all three), composable (combine .btn + .btn-primary), and have mid-level specificity.

ID Selector — #id-name {}

#unique-hero — This element has an ID selector applied. There should only ever be one element with this ID on the page.

⚠️ ID selectors have very high specificity, making them hard to override. Prefer classes for styling. Reserve IDs for JavaScript hooks or anchor links (href="#section").

Universal Selector — * {}

span em strong code

The universal selector * targets everything. Most commonly used in the global reset: *, *::before, *::after { box-sizing: border-box; }. Avoid for general styling — it's too broad.

3 — Color Systems

Named Colors

tomato
steelblue
mediumseagreen
goldenrod
mediumpurple

147 named colors exist in CSS. Great for quick prototyping or teaching. Avoid in production — hard to manage precise branding through names alone.

HEX Colors — #RRGGBB

#1a1a2e
#e94560
#0f3460
#f5f5f5
#16213e

HEX is the most common format in design tools (Figma, etc.). Use #RRGGBBAA for transparency (last 2 digits = alpha). Example: #e9456080 = 50% transparent red.

RGB / RGBA — rgba(r, g, b, alpha)

rgb(231, 76, 60)
rgba(231, 76, 60, 0.7)
rgba(231, 76, 60, 0.4)
rgba(231, 76, 60, 0.1)

Use rgba() when you need transparency — overlays, semi-transparent backgrounds, shadow tints. Alpha is 0 (invisible) to 1 (fully opaque).

HSL / HSLA — hsl(hue, saturation%, lightness%)

hsl(210,70%,25%)
hsl(210,70%,40%)
hsl(210,70%,55%)
hsl(210,70%,70%)
hsl(210,70%,85%)

HSL shines for building palettes: keep hue and saturation constant, only vary lightness to get shades. Hue is 0–360° (color wheel), saturation 0%=gray→100%=vivid, lightness 0%=black→100%=white.

4 — Typography Properties

font-family — Font Stacks

Serif stack: Georgia, 'Times New Roman', Times, serif — used for editorial, editorial, or long-form reading.

Sans-serif stack: 'Helvetica Neue', Arial, sans-serif — clean, modern UI default.

Monospace stack: 'Courier New', Courier, monospace — for code, data, terminals.

Always end your stack with a generic family: serif, sans-serif, monospace, cursive, or fantasy. The browser uses it as a last resort if all named fonts fail to load.

font-size — Sizing Units

12px — Absolute pixel size. Captions, fine print.

16px — Browser default body text size.

1rem — Same as 16px if root is default. Scales with user prefs.

1.25rem — Slightly larger. Subheadings.

2rem — Section titles.

3rem — Hero headings.

⚠️ Prefer rem over px for font sizes. rem respects the user's browser font-size preferences (accessibility), while px overrides them. Reserve px for borders and fixed-pixel layouts.

font-weight — Light to Black

300 — Light. Elegant, airy headings.

400 — Normal (default). Body text.

500 — Medium. Slightly emphasized UI labels.

600 — Semibold. Navigation, subheadings.

700 — Bold. Strong emphasis, CTAs.

900 — Black/Extra Bold. Impactful display text.

Numeric weights (100–900 in steps of 100) require the loaded font to actually support those weights. If a weight isn't available, the browser picks the nearest one. Google Fonts lets you select specific weights on import.

text-align — Text Alignment

left (default)

Text aligned to the left edge of its container. The standard for body copy in left-to-right languages.

center

Centered text. Use for headings, hero text, card titles, and short labels — not long paragraphs.

right

Right-aligned. Useful for numeric table columns, timestamps, or RTL language support.

justify

Justified text stretches each line to fill the container. Used in print-style layouts and newspapers. Can create awkward spacing gaps in narrow columns.

⚠️ Avoid justify on narrow containers or responsive layouts — it creates uneven word spacing ("rivers of white space") that harms readability.

line-height — Leading / Vertical Rhythm

line-height: 1 (tight)

This text has a line-height of 1, meaning lines are packed tightly. Good only for large display headings where you want dramatic closeness.

line-height: 1.5 ✅ Body text sweet spot

A unitless value of 1.5 means each line is 1.5× the font-size. This is the WCAG accessibility recommendation for body text. Comfortable to read.

line-height: 2 (loose)

A value of 2 creates very open spacing between lines. Sometimes used in educational content, legal text, or UI components that need breathing room.

⚠️ Always use unitless values (e.g., 1.5 not 24px or 150%) for line-height. Unitless values scale proportionally with the font-size of each element. Fixed units can break when nested elements have different font sizes.
