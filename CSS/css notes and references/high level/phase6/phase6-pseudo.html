1 — Interaction State Pseudo-Classes

What Is a Pseudo-Class?

A pseudo-class selects an element based on a state or condition that cannot be expressed by a plain HTML attribute — like "the user's mouse is over this element" or "this checkbox is checked." They always start with a single colon :state.

:hover — Mouse Over

Hover this link

Hover the card

Hidden content revealed on hover ✓

:hover triggers when the pointing device is over an element. Use it for visual feedback — colour changes, underlines, shadows, reveals.

⚠️ Touch devices have no hover state — never hide critical content or actions behind :hover alone. Guard with @media (hover: hover) (Phase 5).

:focus & :focus-visible — Keyboard Focus

Focus me too

:focus fires on any focus — click or keyboard.
:focus-visible fires only when the browser determines a focus ring is useful — keyboard navigation yes, mouse click usually no.

⚠️ Accessibility rule: NEVER write :focus { outline: none; } without providing an alternative visual indicator. Keyboard users depend on the focus ring to know where they are on the page. Use :focus-visible to show the ring only for keyboard users while hiding it for mouse clicks.

:active — Currently Being Clicked

Click & hold link

:active applies while the element is being activated — the moment between mousedown and mouseup. Use it for "press" effects on buttons: a slight downward transform: translateY(1px) and reduced shadow mimics a physical button being pressed.

:disabled & :enabled

:disabled matches any form element with the disabled HTML attribute. Use it to style inactive controls — reduce opacity, change cursor, remove hover effects.

⚠️ pointer-events: none prevents clicks but doesn't make an element truly disabled — it's still in the tab order and accessible. Only the HTML disabled attribute truly disables a form element.

:checked — Selected Checkboxes & Radios

Custom checkbox/radio styling requires hiding the native input with appearance: none (or opacity: 0) and building the visual with a sibling element styled via input:checked + .cb-box. The adjacent sibling combinator (+) selects the element immediately after the input in the DOM.

:link & :visited — Link States

Unvisited link (:link) Visited link (:visited) — if you've been to example.com

The full link state order: LVHA — Link, Visited, Hover, Active. Always write them in this order to avoid specificity conflicts overriding each other.

⚠️ Browsers heavily restrict what CSS can be applied via :visited for privacy — only colour properties are allowed (color, background-color, border-color, outline-color).

:first-child selects an element that is the first sibling among ALL its siblings (regardless of type). :last-child selects the last sibling.

Common pattern: remove the bottom border/margin from the last item in a list so there's no dangling separator — li:last-child { border-bottom: none; }

The An+B formula:
n = 0, 1, 2, 3… (auto-increments)
A = the step (every Ath element)
B = the offset (start counting from Bth)

Practical examples:
:nth-child(2n) → even elements (2, 4, 6…)
:nth-child(2n+1) → odd elements (1, 3, 5…) = odd
:nth-child(3n+2) → 2, 5, 8, 11… (every 3rd from 2nd)
:nth-child(-n+3) → first 3 elements only
:nth-child(n+4) → all elements from 4th onwards

:nth-of-type() — Count Within Type

Heading (h4:first-of-type)

First paragraph (p:nth-of-type(1))

Second paragraph (p:nth-of-type(2)) ← styled

Third paragraph (p:nth-of-type(3))

Second Heading (h4:last-of-type) ← styled

:nth-of-type() counts only elements of the same tag type among siblings. This differs from :nth-child() which counts ALL siblings regardless of type.

Rule of thumb: :nth-child for uniform lists (all <li>). :nth-of-type for mixed content where you need to target by element type independently.

3 — :has() — The Parent Selector

The Game-Changer

For 25 years CSS had no way to style a parent based on its children. :has() changes everything — it selects an element IF it contains a matching descendant. Style the card differently when it has an image. Style the form when it contains an error. Style the nav when it contains the active link.

:has() is also called the "relational pseudo-class." It accepts any complex selector inside.

Real patterns:
form:has(input:invalid) — style a form with errors
label:has(+ input:required) — mark required field labels
li:has(ul) — style list items that have nested sub-lists
.card:has(:hover) — style the card when ANY child is hovered
Browser support: All modern browsers since 2023.

4 — Pseudo-Elements

Pseudo-Classes vs Pseudo-Elements

Pseudo-classes (:single-colon) select elements in special states. Pseudo-elements (::double-colon) create or target parts of an element that don't exist as separate DOM nodes — like the first letter of a paragraph, or injected decorative content.

::before and ::after require content: "" — even if the value is empty — or they won't render at all. The content value can be a string, a URL (for images), a counter, or a CSS variable.

These pseudo-elements are visual only — they are not in the DOM, cannot be selected by JavaScript, and are generally ignored by screen readers (don't put important text here).

::first-line & ::first-letter

Typography in editorial design has always elevated the first letter as a visual anchor — the drop cap. With ::first-letter, CSS can replicate this centuries-old typographic tradition. The browser dynamically determines which characters form the first "letter" so it works regardless of content.

The first line of this paragraph is styled differently via ::first-line. Notice how it automatically reflows as you resize the window — it always targets the first rendered line, not a fixed number of words.

::first-letter only works on block-level elements. It accepts a limited set of properties: font, text, color, background, margin, padding, border, float.

::first-line responds to the actual rendered line break — resize the browser and it adjusts automatically. Only font, text, color, and background properties work on it.

::placeholder & ::selection

::placeholder — style input hint text

::selection — select some text in this paragraph

Select some of this text with your mouse to see a custom selection highlight colour applied via ::selection { background: …; }. Only background and color can be styled on ::selection.

::placeholder accepts most text and colour properties. Keep placeholder text low-contrast and italic to visually distinguish it from user-entered values — but not so low contrast it fails WCAG AA (4.5:1 for normal text).

::selection only accepts color, background-color, and text-shadow. Use a colour that maintains good contrast — selected text must remain readable.
