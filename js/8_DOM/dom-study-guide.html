01
What is the DOM?
💡 Simple analogy: Your HTML file is like a recipe written on paper — just text, can't do anything. The browser is the chef who reads it and sets up all the ingredients on the counter in an organized way. That organized setup on the counter is the DOM — live, interactive, and ready to use.

DOM stands for Document Object Model. When the browser reads your HTML file, it doesn't just display it — it converts it into a collection of objects in memory that JavaScript can read and change.

// Your HTML file — just text
<body>
  <h1>Hello</h1>
  <p>World</p>
</body>

// After the browser reads it, the DOM looks like this in memory:
document
└── body
    ├── h1  → contains text "Hello"
    └── p   → contains text "World"

// Now JavaScript can interact with it:
document.querySelector('h1').textContent = 'Hi!';
// Page updates instantly — HTML file is NOT changed

DOM vs HTML — what's the difference?

HTMLDOM
What is it?A text file you writeA live object in memory
Where does it live?On your computer/serverInside the browser
Can JavaScript change it?✗ No✓ Yes
Updates live?✗ No✓ Yes
// Run this in the browser:
document.querySelector('p').remove();

// Result:
// DOM      → the <p> is gone from the page ✓
// HTML file → still has <p> in it, unchanged ✓
// They are two different things!

What is a Node?

Everything in the DOM is a node. A node is simply one unit in the tree. There are 3 types you need to know:

Type 1
Element Node
Any HTML tag.
<div> <p> <h1> <span>
The most common node type.
Type 2
Text Node
The actual text inside a tag.
<p>Hello</p>
"Hello" is a separate text node.
Type 3
Comment Node
HTML comments are real nodes.
<!-- comment -->
Invisible to users, exists in DOM.
⚠️
When you write <p>Hello</p> that is two nodes — an Element node (<p>) and a Text node ("Hello") inside it. This is why some navigation properties surprise beginners.
Quick Summary
HTML file  →  browser reads it  →  builds the DOM (a tree of nodes)
                                            ↓
                             JavaScript can now read, change,
                             add, or delete anything in that tree
02
How the Browser Builds the DOM

When you open a webpage, the browser goes through these exact steps to turn your HTML text into a usable DOM tree:

1
Bytes → Browser receives raw bytes from the server. e.g. 3C 68 74 6D 6C 3E
2
Characters → Bytes are decoded using UTF-8 into readable characters. e.g. <html>
3
Tokens → Characters are broken into meaningful pieces. e.g. < | html | >
4
Nodes → Each token becomes a node object with properties.
5
DOM Tree → Nodes are linked together into the parent-child tree structure.
6
Ready → JavaScript can now fully interact with the tree.
ℹ️
The browser builds the DOM incrementally — it doesn't wait for the full HTML. It starts immediately, token by token. This is why large pages can display partially before fully loading.

Why does <script> go at the bottom?

When the browser hits a <script> tag while building the DOM — it stops everything and runs the script first, because JavaScript might try to modify the DOM while it's being built.

❌ Bad — script runs before DOM is built, can't find elements
<head>
  <script src="app.js"></script>
</head>
<body>
  <h1 id="title">Hello</h1>
</body>

✓ Good — DOM is fully built before the script runs
<body>
  <h1 id="title">Hello</h1>

  <script src="app.js"></script>  ← always at the bottom
</body>
03
Tree Structure & Navigation
💡 Think of the DOM like a family tree. Every element has parents, children, and siblings — and JavaScript gives you properties to move between all of them.

The HTML we'll use for all examples

<body>
  <div id="parent">
    <!-- a comment -->
    <h1>Title</h1>
    <p>Paragraph</p>
    <span>Span</span>
  </div>
</body>

This is how the DOM tree actually looks for that HTML:

document └── body └── div#parent ├── #comment "a comment" ├── h1 │ └── #text "Title" ├── p │ └── #text "Paragraph" └── span └── #text "Span"

The Golden Rule

Always use the Element version of every property. The plain versions (childNodes, firstChild, etc.) catch invisible whitespace text nodes and comments between tags — almost never what you want. The Element versions skip all of that.

Going UP — to the parent

const h1 = document.querySelector('h1');

h1.parentNode;     // → <div id="parent">  (returns ANY node type)
h1.parentElement;  // → <div id="parent">  (returns only Elements)

// The difference only shows at the very top:
document.documentElement.parentNode;     // → #document
document.documentElement.parentElement;  // → null ← parent is not an Element

Going DOWN — to the children

const parent = document.getElementById('parent');

parent.childNodes;
// → [#comment, #text, h1, #text, p, #text, span, #text]
// ↑ sees EVERYTHING including whitespace text nodes

parent.children;
// → HTMLCollection [h1, p, span]
// ↑ sees ONLY elements — use this ✓

parent.firstChild;         // → #comment  (the comment node!)
parent.firstElementChild;  // → <h1>  ← use this ✓

parent.lastChild;          // → #text (whitespace after </span>)
parent.lastElementChild;   // → <span>  ← use this ✓

Going SIDEWAYS — to siblings

const h1 = document.querySelector('h1');

h1.nextSibling;              // → #text (whitespace between </h1> and <p>)
h1.nextElementSibling;       // → <p>  ← use this ✓

const p = document.querySelector('p');
p.previousSibling;           // → #text (whitespace)
p.previousElementSibling;    // → <h1>  ← use this ✓

Full navigation example

const parent = document.getElementById('parent');

const h1   = parent.firstElementChild;             // <h1>
const p    = h1.nextElementSibling;                // <p>
const span = p.nextElementSibling;                 // <span>
const back = span.previousElementSibling;          // <p>
const up   = span.parentElement;                   // <div#parent>

// Entire tree navigated — no querySelector needed!

04
Selecting Elements
💡 Like searching for a book in a library — you can search by its unique ID number (fastest), by category, by type, or by a full description (most flexible).

The HTML we'll use for all examples

<h1 id="main-title">Hello</h1>
<p class="text">First</p>
<p class="text special">Second</p>
<p>Third — no class</p>
<div class="box">
  <span class="text">A span</span>
</div>

1. getElementById

const title = document.getElementById('main-title');
// → <h1 id="main-title">Hello</h1>

document.getElementById('nothing');
// → null
✓
Fastest selector — direct lookup. Returns one element or null. Only available on document, not on sub-elements.

2. getElementsByClassName

const items = document.getElementsByClassName('text');
// → HTMLCollection [<p>, <p>, <span>] — all 3 elements with class "text"

// Can be used on a specific element too:
const box = document.querySelector('.box');
box.getElementsByClassName('text');
// → HTMLCollection [<span>] — only finds .text INSIDE .box
ℹ️
Returns a live HTMLCollection — automatically updates when the DOM changes.

3. getElementsByTagName

const paragraphs = document.getElementsByTagName('p');
// → HTMLCollection [<p>, <p>, <p>] — all paragraphs

document.getElementsByTagName('*');
// → ALL elements on the page

4. querySelector

document.querySelector('p');              // first <p>
document.querySelector('.text');          // first element with class "text"
document.querySelector('.text.special');  // must have BOTH classes
document.querySelector('#main-title');    // by ID
document.querySelector('.box span');      // span INSIDE .box
document.querySelector('.nothing');      // → null if not found
✓
Accepts any valid CSS selector. Most flexible. Returns the first match or null.

5. querySelectorAll

document.querySelectorAll('p');
// → NodeList [<p>, <p>, <p>]

document.querySelectorAll('.text');
// → NodeList [<p>, <p>, <span>]

document.querySelectorAll('p.text');
// → NodeList [<p>, <p>] — only <p> elements that also have class "text"

document.querySelectorAll('.nothing');
// → empty NodeList [] — NOT null

Live vs Static — the most important difference

const live   = document.getElementsByClassName('text'); // HTMLCollection — LIVE
const frozen = document.querySelectorAll('.text');          // NodeList — STATIC

console.log(live.length);    // 3
console.log(frozen.length);  // 3

// Add a new element with class "text"
const newP = document.createElement('p');
newP.className = 'text';
document.body.appendChild(newP);

console.log(live.length);    // 4 ← updated automatically!
console.log(frozen.length);  // 3 ← still frozen at the old count
Live (window)
HTMLCollection
Like a window looking at the DOM — always shows current state. Returned by getElementsByClassName and getElementsByTagName.
Static (photo)
NodeList
Like a photograph of the DOM — frozen at the moment of the query. Returned by querySelectorAll.

Are they arrays? No — but you can convert them

const items = document.querySelectorAll('.text');

❌ items.map(...)    → error, not a real array
❌ items.filter(...) → error

✓ items.forEach(item => console.log(item.textContent));
✓ Array.from(items).map(item => item.textContent);

// HTMLCollection does NOT support forEach — must convert first:
const coll = document.getElementsByClassName('text');
✓ Array.from(coll).forEach(item => console.log(item.textContent));

05
Manipulating Elements
💡 Think of a DOM element like a physical box. It has a label (the text), stickers (the classes), written properties (attributes like src, href), and a color (the style). Manipulation means changing any of these after the page loads.

The HTML we'll use for all examples

<h1 id="title">Hello <strong>World</strong></h1>
<p id="para" class="text old-style">A paragraph</p>
<img id="photo" src="cat.jpg" alt="A cat">

Part 1 — Text Content

Safe · Fast
textContent
Plain text only. Ignores all HTML tags. XSS-safe. Use this most often.
CSS-aware
innerText
Like textContent but respects CSS. Hidden elements are excluded from the result.
Renders HTML
innerHTML
Gets/sets actual HTML markup. Powerful but dangerous with user input.
// HTML: <h1>Hello <strong>World</strong></h1>
const h1 = document.getElementById('title');

// GETTING:
h1.textContent;  // "Hello World"                      — plain text, ignores tags
h1.innerText;    // "Hello World"                      — same here
h1.innerHTML;    // "Hello <strong>World</strong>"     — returns tags too

// SETTING:
h1.textContent = '<em>test</em>';  // shows literally: <em>test</em>
h1.innerHTML = '<em>test</em>';   // renders italic: test

The hidden difference — CSS visibility

// HTML: <div>Visible <span style="display:none">Hidden</span></div>
const box = document.querySelector('div');

box.textContent;  // "Visible Hidden"  → returns ALL text, even hidden
box.innerText;    // "Visible"         → respects CSS, hidden text excluded
⚠️
Never use innerHTML with user input — it can execute malicious scripts. Always use textContent when setting text from user input.
// ❌ Dangerous — XSS attack vector
element.innerHTML = userInput;

// ✓ Safe — always use for user-provided text
element.textContent = userInput;

Part 2 — Attributes

const img = document.getElementById('photo');

// Read
img.getAttribute('src');   // "cat.jpg"
img.getAttribute('alt');   // "A cat"
img.getAttribute('width'); // null — doesn't exist

// Write / Create
img.setAttribute('src', 'dog.jpg');   // changes the image
img.setAttribute('width', '300');     // creates new attribute

// Delete
img.removeAttribute('alt');            // removes it entirely

// Shortcut properties for common attributes:
img.src;   // same as getAttribute('src')
img.alt;   // same as getAttribute('alt')
img.id;    // same as getAttribute('id')

Part 3 — Classes

const para = document.getElementById('para');
// currently: class="text old-style"

para.classList.add('active');
// → class="text old-style active"

para.classList.remove('old-style');
// → class="text active"

para.classList.toggle('active');
// → "active" existed → now removed: class="text"

para.classList.toggle('active');
// → "active" was absent → now added: class="text active"

para.classList.contains('text');    // → true
para.classList.contains('nothing'); // → false
✓
toggle is perfect for buttons that switch between two states — dark mode, open/close menus, active/inactive tabs.

Part 4 — Styling

❌ Avoid — inline styles, hard to maintain
para.style.color = 'red';
para.style.fontSize = '24px';         // camelCase in JS!
para.style.backgroundColor = 'yellow'; // not background-color

✓ Better — toggle a CSS class
para.classList.add('highlighted');
// then in style.css: .highlighted { color: red; font-size: 24px; }
ℹ️
CSS property names become camelCase in JavaScript: background-color → backgroundColor, font-size → fontSize, margin-top → marginTop.

Exception: Use inline styles when the value is dynamic (comes from a variable or calculation): 
element.style.width = progress + '%';   // value changes at runtime ✓
element.style.top   = mouseY + 'px';   // depends on mouse position ✓
Full Summary — All Manipulation
TEXT:
  textContent  → plain text, safe, fast ← use this most
  innerText    → plain text, respects CSS visibility
  innerHTML    → renders HTML ← never use with user input

ATTRIBUTES:
  getAttribute('x')       → read attribute value
  setAttribute('x', 'v')  → write or create attribute
  removeAttribute('x')    → delete attribute entirely

CLASSES:
  classList.add('x')       → add class
  classList.remove('x')    → remove class
  classList.toggle('x')    → flip — adds if absent, removes if present
  classList.contains('x')  → check — returns true or false

STYLING:
  ✓ classList.add('my-class')  ← always prefer this
  ✓ style.property = value     ← only when value is dynamic
