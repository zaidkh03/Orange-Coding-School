Modern Layout Systems

Flexbox for one-dimensional flows. Grid for two-dimensional structures. Subgrid for alignment across nested components. These three tools replace every float hack ever written.

Part A

Flexbox — One-Dimensional Layout

Flexbox distributes space along a single axis (row or column). Perfect for nav bars, button groups, cards in a row, and any component whose items need to align, space, and wrap intelligently.

1 — display: flex & flex-direction

display: flex — Activating Flexbox

Setting display: flex on a container makes it a flex container. Its direct children automatically become flex items. The container controls alignment and distribution; items control their own growth and shrinking.

flex-grow distributes leftover space — not the total width. If items have natural sizes of 80px each in a 400px container, there's 160px of leftover. grow: 1 on the middle item gives it all 160px. grow: 1 on all three splits the 160px equally (~53px each).

flex-basis overrides width (in row direction). The shorthand flex: 1 expands to flex: 1 1 0 — grow=1, shrink=1, basis=0. With basis=0, all space is distributed from scratch (ignoring content size), which gives truly equal columns. flex: 1 1 auto distributes extra space, so larger items stay proportionally larger.

CSS Grid — Two-Dimensional Layout

Grid places items into explicit rows AND columns simultaneously. Use it for full page layouts, dashboard grids, magazine-style compositions — anywhere you need control in both dimensions at once.

The fr unit (fractional) distributes remaining space after fixed sizes are satisfied. 1fr 2fr 1fr gives the middle column twice as much space as the others.

repeat(auto-fill, minmax(160px, 1fr)) is the most powerful one-liner in CSS. It creates as many columns as can fit at 160px minimum, each growing to fill available space — giving you a responsive grid with zero media queries.

auto-fill creates empty tracks if possible. auto-fit collapses empty tracks — items stretch more.

grid-column: span 2 means "occupy 2 column tracks". grid-column: 1 / 3 means "start at line 1, end at line 3" (same result on a 4-column grid). grid-column: 1 / -1 means "start at line 1, end at the last line" — full-width regardless of column count. Very useful.

grid-template-areas — Named Regions

ASCII-art layout — the most readable way to define a page structure

header grid-area: header
sidebar grid-area: sidebar
main grid-area: main
aside grid-area: aside
footer grid-area: footer

grid-template-areas uses quoted strings to define a visual map of your layout. Each quoted string = one row. Repeated names span across cells. A dot (.) = empty cell.

The CSS reads like an ASCII diagram — immediately obvious to any developer reading it. This is its biggest advantage over line-number placement.

Subgrid lets a child grid element inherit its parent's track definitions instead of creating new ones. Classic use case: a row of cards where titles, descriptions, and buttons must align horizontally across cards — even when content length varies.

The parent defines the row tracks. Each card uses grid-row: span 4 and grid-template-rows: subgrid to participate in those shared tracks. Browsers: all modern browsers since 2023.
