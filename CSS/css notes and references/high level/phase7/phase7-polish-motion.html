Shorthand order: border-radius: TL TR BR BL (clockwise from top-left). Elliptical syntax: border-radius: 40% 60% / 60% 40% — horizontal radii / vertical radii.

border-radius: 50% on a square makes a circle. On a non-square it makes an ellipse. border-radius: 9999px makes a pill shape regardless of element dimensions — safer than 50% for rectangles.

Syntax: box-shadow: offset-x offset-y blur spread color
— offset-x/y: position (negative = left/up)
— blur: softness (0 = hard edge)
— spread: size change before blurring (negative shrinks)
— inset: keyword before values = inner shadow

Design tip: Use spread 0 and a tinted shadow colour (not just black) for realistic depth. Layer 2–3 shadows with increasing blur for natural fall-off.

filter applies to the entire element including its children. Multiple filters chain with spaces: filter: blur(2px) brightness(1.2) saturate(1.5).

filter: drop-shadow() differs from box-shadow — it follows the alpha shape of the element (works on PNGs with transparency). box-shadow always follows the rectangular bounding box.

backdrop-filter — Frosted Glass Effect

Frosted Glass Panel

backdrop-filter: blur(16px) saturate(1.8)

The coloured blobs behind this panel are blurred — only the background is affected, not this text.

backdrop-filter blurs/filters what is BEHIND the element. The element itself must have some transparency (background: rgba(…, 0.x)) for the effect to show through.

⚠️ backdrop-filter creates a new stacking context and can be expensive on GPU. Limit to key UI surfaces like modals, nav bars, and sidebars. All modern browsers support it.

Chained transforms — applied right to left

translate + rotate + scale

transform does NOT affect document flow — the element's original space is preserved. Nearby elements don't reflow. This makes transforms the right tool for animations and hover effects.

Multiple transforms chain with spaces and apply right to left: transform: translate(20px) rotate(45deg) rotates first, then translates — not what you might expect.

transform-origin sets the anchor point for rotation and scaling. Default is 50% 50% (center). For a rotating gear icon, set it to center. For a fold-out card, set to an edge.

translate, rotate, scale — Individual Properties

translate: 0 -10px
rotate: 45deg
scale: 1.2

CSS now has individual transform properties: translate, rotate, scale (not inside transform:). They apply in a fixed order (translate → rotate → scale) and can be animated independently — you can transition only rotate without specifying translate and scale in the keyframe.

These are newer (Chrome 104+, Firefox 72+, Safari 14.1+) and ideal for complex multi-property animations where you need independent control.

transition: multiple properties — separate with comma

Shorthand: transition: property duration timing-function delay
transition: all 0.3s ease — transitions ALL animatable properties.
transition: color 0.2s ease, transform 0.3s ease-out — different durations per property.

Easing guide:
— ease-out: fast start, gentle stop — best for elements entering the screen.
— ease-in: slow start, fast finish — best for elements leaving the screen.
— ease-in-out: both — best for back-and-forth toggle animations.
— linear: constant speed — good for looping animations, bad for UI interactions.

⚠️ transition: all is convenient but transitions EVERY changed property — including ones like width that are expensive to animate. Be explicit.
